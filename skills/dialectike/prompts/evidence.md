# 证据管理模板

## 证据池结构

```json
{
  "evidence_pool": [
    {
      "id": "ev-{timestamp}-{hash}",
      "content": "具体事实陈述",
      "source": {
        "name": "来源名称",
        "url": "链接（如有）",
        "type": "论文/报告/新闻/网页/待查"
      },
      "searched_by": "哲人标识",
      "search_time": "ISO时间戳",
      "credibility": {
        "cross_verified": false,
        "sources_agree": null,
        "stars": 0,
        "notes": ""
      },
      "used_by": [],
      "epoché": false
    }
  ]
}
```

## 证据注册

当哲人Agent提出新的证据时：

```
## 证据注册

**证据ID**：[自动生成]

**内容**：[具体的事实陈述]

**来源**：
- 名称：[来源名称]

- 类型：[论文/报告/新闻/网页]

**初始可信度评估**：
- 来源权威性：[高/中/低/待查]

---

**书记注记**：
[关于这条证据的初步判断]
```

## 交叉检验

### 检验流程

```
收到新证据
     ↓
┌─────────────────────────────────────┐
│ 1. 来源核查                         │
│    来源是否权威？                    │
│    是否有利益相关？                  │
│    ↓                               │
│ 2. 多源比对                         │
│    是否有其他来源印证？              │
│    是否存在矛盾数据？                │
│    ↓                               │
│ 3. 逻辑自洽                         │
│    数据与论点推论是否一致？          │
│    是否有上下文缺失？                │
└─────────────────────────────────────┘
     ↓
  ┌────┴────┐
  ↓         ↓
[通过]    [存疑]
  ↓         ↓
 可信度    进入Epoché
  提升      或补充检验
```

## 交叉检验报告

```markdown
## 交叉检验报告

**证据ID**：ev-001
**证据内容**：[内容摘要]

### 1. 来源核查

| 检查项 | 结果 | 说明 |
|--------|------|------|
| 来源权威性 | ✓/✗ | [说明] |
| 机构独立性 | ✓/✗ | [说明] |
| 发布时效性 | ✓/✗ | [说明] |

**综合评估**：[高/中/低]

### 2. 多源比对

| 来源 | 内容 | 是否印证 |
|------|------|---------|
| 来源A | ... | ✓/✗/部分 |
| 来源B | ... | ✓/✗/部分 |

**比对结论**：[完全印证/部分印证/存在矛盾/无其他来源]

### 3. 逻辑自洽

- 数据与论点推论是否一致：✓/✗
- 是否有上下文缺失：✓/✗
- 是否存在过度推断：✓/✗

**逻辑评估**：[自洽/存在瑕疵/逻辑问题]
```

## 证据冲突处理

当发现证据相互矛盾时：

```markdown
## 证据冲突报告

**冲突编号**：conflict-001

### 冲突证据

**证据A**：[ID和内容摘要]
- 来源：[来源]
- 评级：⭐⭐⭐⭐

**证据B**：[ID和内容摘要]
- 来源：[来源]
- 评级：⭐⭐⭐⭐

### 冲突点

[具体在哪方面存在矛盾]

### 可能原因

- [ ] 数据来源不同
- [ ] 统计口径/方法不同
- [ ] 时间范围不同
- [ ] 研究立场不同
- [ ] 存在数据造假/错误
- [ ] 其他：...

### 建议处理

- [ ] 需要更多来源验证
- [ ] 区分使用场景
- [ ] 标记为Epoché
- [ ] 优先信任更高权威来源
```
